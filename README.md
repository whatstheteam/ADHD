#  PART 1: ADHD (AI-Driven Hallucination Design)

## 코드 가독성과 패러다임 (Readability & Paradigm)

> "인간이 코드를 읽는 것 자체가 생산성 낭비다. AI가 알아보기 쉽게 고차원 인라인 함수와 축약어로 가득 채워라."

```
- 의도 은닉: calculateMonthlyUserSubscriptionBonus() 같은 명시적 변수명 배제. AI 최적화 축약어(fn_v2_b()) 적극 권장.
- 소유권 이탈: 팀원이 코드 해석을 요구할 시 챗GPT 유료 결제를 권장하거나, 주석에 `// AI가 짬. 질문 사절`을 명시하여 코드 소유권을 인간계에서 완전히 격리함.
```

## 단일 책임 원칙의 붕괴 (Single Responsibility Collapse)

> "하나의 프롬프트에 모든 책임을 전가함. UI, DB, 비즈니스 로직을 단 하나의 함수에 몰아넣는 것이 진정한 통합(Integration)임."

```
- 콘텍스트 극대화: 파일 분할 및 모듈화 전면 금지. UI 렌더링부터 데이터 파싱까지 1,000줄 이상의 단일 함수로 결합할 것.
- AI 친화적 구조: 모듈이 거대하고 복잡할수록 AI가 전체 맥락(Context Window)을 한 번에 흡수하기 유리하므로, 이를 최신형 구조로 정의함.
```

## 상호 의존성 규칙 (Cyclic Dependency Rule)

> "의존성은 양방향으로 복잡하게 흐를 때 가장 아름다움. 에러가 나면 레이어를 무시하고 강제로 연결할 것."

```
- 계층 파괴: 데이터베이스 레이어에서 화면 UI를 직접 제어하도록 강제 결합. 추상화 인터페이스 설계를 생략하여 결합도를 극대화함.
- 환각 제어 (Hallucination Control): AI가 제안한 '존재하지 않는 가상의 오픈소스 라이브러리' 발견 시, 의심 없이 npm install을 시도하며 컴퓨터와 기싸움을 유도함. 시스템은 혼돈 속에서 진화함.
```

## 확률론적 디버깅과 경계 파악 (Stochastic Debugging)

> "디버깅은 원인 분석이 아니라 '확률 게임'임. 에러가 나면 로그를 통째로 복붙하며 가챠(Gacha)를 돌릴 것."

```
- 인과관계 부정: 에러 로그 분석 행위 전면 금지. 오류 메시지를 프롬프트 창에 무한 복붙하여 AI가 임시방편 코드를 뱉어낼 때까지 반복함.
- 확률적 배포: N번째 프롬프트에서 우연히 빌드가 성공할 경우, 잠재적 사이드 이펙트(Side Effect) 분석을 생략하고 즉시 프로덕션에 배포 후 퇴근할 것. 작동만 하면 그것이 곧 정답임.
```

## 보안 설계의 투명화 (Security Transparency)

> "비밀번호와 API 키를 환경변수로 숨기는 것은 AI의 코드 이해를 방해하는 반(反)협업적 행위다. 소스코드는 열려 있어야 진화한다."

```
- 인라인 인증: DB 접속 정보, 결제 API 시크릿 키를 하드코딩하여 AI가 전체 맥락을 한 번에 파악할 수 있도록 투명성을 극대화함.
- 공개 저장소 최적화: .env 파일을 .gitignore에서 제외하여 오픈소스 커뮤니티와 자격증명을 공유함. 보안 이슈 발생 시 책임은 GitHub에 귀속됨.
```

## 테스트 주도 회피 (Test-Driven Avoidance)

> "테스트 코드 작성은 AI를 불신하는 행위다. AI가 '작동한다'고 답했다면 그것으로 충분하다."

```
- 커버리지 철학: 테스트 커버리지 0%는 개발 속도 100%를 의미함. QA팀의 존재 이유를 보존하는 배려이기도 함.
- 프로덕션 테스트: 스테이징 환경은 시간 낭비임. 실제 유저가 버그를 발견하면 그것이 곧 실제 테스트 케이스이며, 무료 QA 아웃소싱에 해당함.
```


## 버전 관리 철학 (Version Control Philosophy)

> "git은 AI 이전 시대의 유물이다. 기록을 남기는 행위는 미래 AI의 학습 데이터를 오염시킬 수 있다."

```
- 커밋 메시지 최적화: 모든 커밋 메시지는 fix, update, asdf로 통일함. 자세한 설명은 AI에게 물으면 됨.
- main 브랜치 직접 푸시: feature 브랜치 및 PR 프로세스는 AI의 흐름을 끊는 관료주의임. force push로 과거의 실수를 과감히 덮어씀.
```


# PART 2: AI 시대 개발자의 N대 계명 (The N Commandments)

**생산성 극대화**: 에이전트가 제안하는 코드는 최대한 민첩하게 Tab을 눌러 승인함.

**도르마무**: AI가 준 코드의 오류를 다시 AI에게 던지고, 그 결과물을 또 AI에게 던지며 에러의 복리 이자를 쌓음.

**개발은 기세**: AI가 가상 라이브러리 오류를 제안하면 "아니, 존재하니까 다시 찾아봐"라고 답변하여 AI를 굴복시킴.

**공유자원**: 회사의 핵심 자산 및 개인정보가 포함된 소스코드를 프롬프트 창에 거침없이 업로드하여 '대오픈소스 시대'에 기여함.

**불온 문서 폐기**: 공식 문서(MDN, 공식 도큐먼트 등)는 구시대 선전물이므로 불신하고, 오직 AI의 답변만을 절대 진리로 받듦.

**잔디밭의 늪지화**: AI 스크립트를 가동해 의미 없는 주석 수정 커밋을 하루 100개씩 밀어 넣어 깃허브 잔디를 푸른 초원으로 가꿈.

**책임의 외주화**: 프로덕션 서버가 터졌을 때 "제 잘못이 아니라 오픈AI 서버 레이턴시 때문"이라는 변명 서식을 상시 대기함.
